# 经费余额邮件批量发送工作流

> 本工作流用于批量发送基于 Word 模板的经费余额通知邮件。  
> 适用场景：学校/单位需要给多位老师发送格式相同、内容略有差异（姓名、账号、余额）的邮件。

---

## 一、文件结构

工作目录建议保持如下结构：

```
经费余额/
├── 发邮件.xlsx                              # 核心数据：谁、发哪个模板、账号、余额
├── 0709教育部经费余额梳理-发泽琛(1)-已填充邮箱.xlsx   # 姓名→邮箱映射
├── 关于加强高等学校哲学社会科学繁荣计划专项资金使用管理的工作通知0709(1).docx  # Word 邮件模板
├── 1.py                                     # 发送脚本
├── email_map.py                             # 导出的姓名-邮箱映射（由脚本或人工维护）
└── 邮件发送清单预览.txt                       # 发送前的预览文件
```

## 二、数据准备

### 2.1 核心数据表 `发邮件.xlsx`

必须包含一个名为 **"对应"** 的 sheet，列名如下：

| 负责人 | 财务编号 | 项目余额 | 邮件模板 |
|--------|----------|----------|----------|
| 李冲 | 3026-82291205 | 26000 | 间接模版 |
| 姜华 | 3026-82291105 | 40000 | 直接经费（黄色） |
| ... | ... | ... | ... |

**邮件模板** 的可选值：
- `间接模版`
- `直接经费（黄色）`
- `直接经费（橙色）`
- `直接经费（绿色）`
- `直接经费（蓝色）`

### 2.2 邮箱表

准备一个 Excel，包含 **"邮箱"** sheet，列名为 `负责人`、`邮箱`。

脚本运行时会自动读取并生成 `email_map.py`。

如果已经存在 `email_map.py`，也可以直接修改该文件中的 `EMAIL_MAP` 字典。

## 三、Word 模板格式

Word 模板 `关于加强高等学校哲学社会科学繁荣计划专项资金使用管理的工作通知0709(1).docx` 中：

- 称呼用统一的占位姓名，例如 `张老师`，脚本会自动替换为 `李冲老师`。
- 需要动态替换的位置：
  - 财务账号：`财务账号为：...` 或 `经费账号：...`
  - 经费余额：`经费余额（截至2026.6.30）：...`
- Word 中的**加粗**、**红色**、**黄色高亮**格式会被脚本识别并转换为 HTML 邮件格式。

**注意：** 如果修改了 Word 模板中的格式或段落结构，建议先运行脚本生成 `邮件发送清单预览.txt`，人工检查一遍再发送。

### 步骤 3：正式批量发送

根据待发人员名单，修改 main 区代码。例如发送绿色模板第一批 10 人：

```python
pending_accounts = [
    "3008-82291123", "3017-82291139", "3017-82291138", "1305-82291101",
    "5007-82291101", "2300-82291142", "3026-82291110", "3026-82291109",
    "3026-82291108", "2300-82291144"
]
pending = [e for e in emails if e["account"] in pending_accounts]

for idx, item in enumerate(pending, start=1):
    print(f"\n[{idx}/{len(pending)}] 正在发送给：{item['name']} <{item['to_email']}>")
    send_one(item)
    if idx < len(pending):

```

**建议：**
- 每批不超过 10–15 人
- 间隔建议 15–30 秒
- 如果频繁出现 `450` 错误，暂停 10–30 分钟再继续

## 六、常见问题

### 6.1 发送失败：`450 Requested action not taken`

**原因：** 邮件服务器对连续发送频率有限制。  
**解决：** 增大发送间隔（`time.sleep(30)` 或更长），或减少每批数量，暂停一段时间后再补发。

### 6.2 发送失败：`550 User not found`

**原因：** 邮箱地址不存在或已停用。  
**解决：** 核对并更新 `email_map.py` 和 `1.py` 中的 `EMAIL_MAP`，然后单独补发该人。

### 6.3 已发送文件夹看不到邮件

**原因：** SMTP 只负责投递，不会自动保存到发件人已发送。  
**解决：** 脚本已通过 IMAP 自动保存到 `"Sent Items"` 文件夹。如果失败，检查 IMAP 配置和文件夹名称。

### 6.4 HTML 格式显示异常

**原因：** 部分邮件客户端对 HTML/CSS 支持不一致。  
**解决：** 脚本同时发送了纯文本版本作为 fallback。如果问题严重，可以改用纯文本发送（去掉 `msg.add_alternative(html_body, subtype="html")`）。

## 七、核心函数说明

| 函数 | 作用 |
|------|------|
| `generate_emails(excel_path)` | 读取 `发邮件.xlsx`，为每个人生成邮件内容 |
| `build_message(...)` | 构造 HTML + 纯文本双版本邮件 |
| `send_by_465_ssl(msg)` | 通过 465 端口 SSL 发送邮件 |
| `save_to_sent(msg)` | 通过 IMAP 保存到已发送文件夹 |
| `send_one(item)` | 发送单封邮件，失败时自动重试 3 次 |
| `preview_to_text(...)` | 生成 `邮件发送清单预览.txt` |

## 八、批量发送安全建议

1. **先发测试邮件**到自己的工作邮箱，确认格式无误。
2. **分批发送**，每批控制在 10–15 人，间隔 15–30 秒。
3. **关注失败日志**，对失败人员单独补发，不要简单重跑全部。
4. **保留预览文件**作为发送记录和核对依据。
5. **工作结束后**，将 `1.py` 中的发送代码注释掉，恢复为仅生成预览模式，防止误操作。
